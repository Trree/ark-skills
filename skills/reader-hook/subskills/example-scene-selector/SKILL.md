# Example Scene Selector

这是 `reader-hook` 的内部 subskill。

## 解决什么问题

当例子出现这些问题时，用这个 subskill：

- 例子只是任务名，不是故事
- 例子太窄，只对少数人有效
- 例子没有代价，没有情绪，没有反差

## 选例子标准

一个可用的例子，至少满足下面 4 条里的 3 条：

- 有代价：返工、尴尬、误判、白费时间
- 有反差：明明处理过，结果又来一遍
- 有情绪：烦、难受、懊恼、泄气
- 能自然引出后面的判断

## 不要这样选

- 选一个专业动作：review、debug、release
- 选一个太泛的状态：最近很焦虑、效率不高
- 选一个只是“有点烦”的情况

## 要这样选

- 回看聊天记录，发现自己一直在丢资产
- 明明已经想明白过，过几天又得从头来
- 同一种错修过一次，后来还是再犯

## 一句话原则

不要找“任务类型”，要找“失败瞬间”。
