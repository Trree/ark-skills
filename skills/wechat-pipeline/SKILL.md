# WeChat Pipeline

这条 pipeline 只做一件事：

**把"作者亲历的一个真实观察"，提炼成"能改变读者某个行为的好文章"。**

两个端点缺一不可。没有真实素材，文章写空。没有行为改变，文章没写完。

## 好内容的五个判断标准

每篇文章在进入写作前，必须能回答以下五条。缺任何一条，退回对应阶段。

1. **素材源头**：这篇文章的核心判断，来自作者的什么具体经历、失败或反常发现？（不是"来自思考"）
2. **翻案句**：很多人以为 ___，但实际上 ___（具体，不是泛泛反转）
3. **不同意句**：大多数读者读到这句话会感到不适或想反驳，但无法否认：___
4. **核心判断句**：1 句，有立场，能独立成句，不是口号
5. **行为改变句**：读者读完这篇之后，___ 这个行为会改变或停止

这五条是内容核心，在阶段 2.5 强制生成，不能为空，不能模糊。

## 链路选择

**品牌深度链路**（默认）：

适用于：需要研究深度、品牌积累、长期读者信任的文章。

```
素材采集 → 品牌收束 → 选题锁定 → 研究 → 内容核心锁定 → 写作
→ 写后审查 → 散文审查 → 格式优化 → 入口优化 → 完成确认 → 分发准备 → [发布]
```

**公众号快写链路**：

适用于：时效性强、表达驱动、素材来自作者当下直接经历的文章。

```
素材采集 → 品牌收束 → 选题锁定 → 大纲 → 内容核心锁定 → 写作
→ 写后审查 → 散文审查 → 格式优化 → 入口优化 → 完成确认 → 分发准备 → [发布]
```

切换到快写链路的条件：

- 用户明确要求走快写链路
- 主题是时效事件、反应型内容、个人观察记录
- 素材来自作者真实经历，不依赖外部研究

两条链路的素材要求、品牌约束、内容核心锁定、写后审查要求完全相同。

## 强制规则

1. **素材采集不可跳过**。作者必须先给出真实素材来源，"来自思考"不算，不能进入选题。
2. **品牌收束不可跳过**。必须确认支柱归属和使命类型，主题无张力则停止推进。
3. **内容核心锁定不可跳过**。五行内容核心必须全部写出，任何一行为空或模糊，退回阶段 2。
4. **写后审查门槛为 16 分**。低于 16 分不进入散文审查，退回修稿。
5. **散文审查不可跳过**。每篇初稿必须经过段落级散文质量检查，不只检查结构。
6. **完成确认不可跳过**。必须能具体回答"读者行为改变"才算完成，否则退回修稿。
7. **入口优化默认执行**。每篇在格式优化后默认过一遍 `reader-hook`。
8. **每次只推进一个阶段**。产物文件不存在，不允许进入下一阶段。
9. **优先复用已有产物**。除非用户明确要求重写。
10. **发布需要用户明确确认**。做到 HTML 产物后暂停，不自动发布。

## 执行流程

### 阶段 -1：素材采集（强制）

**在任何选题、研究、写作开始之前执行。**

目标：确认这篇文章的判断来自作者的真实经历，不是从概念出发。

强制回答以下问题（必须具体，不接受泛泛回答）：

1. **触发事件**：什么具体的经历、失败或反常发现，让你觉得这件事值得写？
   - 合格答案示例：上周我连续三天用 AI 写稿，发现我越来越难判断哪句话是我说的，哪句是它补的
   - 不合格答案：我觉得 AI 会影响判断力

2. **反常点**：这个经历里，什么地方跟你预期的不一样？
   - 合格答案示例：我以为我在用 AI 提效，但我发现我实际上在用 AI 填满我没想清楚的地方
   - 不合格答案：AI 有时候会产生依赖

3. **未解决的困惑**：到现在你还没完全想清楚的是什么？
   - 合格答案示例：我不知道这是我的问题还是工具的设计问题，但我感觉不太对

如果以上三个问题无法具体回答，停止推进。先积累素材，不要开始写作。

输出：`<workspace_root>/00_material/<slug>.md`，记录以上三个问题的答案。

如果用户未提供 `slug`，在此阶段自动生成。

---

### 阶段 0：品牌收束（强制）

使用 `ai-workflow-brand`。

输入：`<workspace_root>/00_material/<slug>.md`

目标：

1. 确认主题落在四大支柱之一（Judgment / Method / System / Brand Asset）
2. 选定文章使命类型（Define / Distinguish / Diagnose / Build / Sharpen）
3. 确认主题有"被误判"或"被低估"的张力，不是工具盘点、资讯搬运或空泛感悟

如果无法完成以上三项，停止推进，重新讨论主题方向。

输出：确认的主题方向、支柱归属、使命类型，写入状态文件。

---

### 阶段 1：选题锁定

品牌深度链路使用 `keyword-research`（不可用则手动完成）。
公众号快写链路跳过关键词研究，直接进入阶段 2。

输入：阶段 0 确认的主题方向 + `00_material/<slug>.md`

输出：`<workspace_root>/01_topics/<slug>.md`

最低要求：

1. 给出 5-10 个候选选题，每个注明与品牌支柱的对应关系
2. 每个选题包含：潜在翻案角度、目标读者困惑、预期核心判断方向
3. 收敛为 1 个最终选题，说明与素材的连接关系

未选用的候选选题追加到 `01_topics/_backlog.md`，不丢弃。

---

### 阶段 2：研究 / 大纲

品牌深度链路使用 `seo-content-brief`（不可用则手动完成）。
公众号快写链路使用 `wechat-topic-outline-planner`（不可用则手动完成）。

输入：`<workspace_root>/01_topics/<slug>.md`

输出：`<workspace_root>/02_research/<slug>.md`

最低要求：

1. 核心问题是什么（读者真实困惑，不是泛泛话题）
2. 大家通常把它理解错在哪里（误判方向）
3. 真实场景候选（至少 2 个，优先来自素材文件中的触发事件）
4. 机制层解释（为什么会误判，平台/工具/认知哪个层面）
5. 结构提纲

---

### 阶段 2.5：内容核心锁定（强制）

使用 `knowledge-article-calibrator`。

输入：`<workspace_root>/02_research/<slug>.md` + `<workspace_root>/00_material/<slug>.md`

输出：`<workspace_root>/03_core/<slug>.md`

必须明确写出以下五行，不能为空，不能模糊：

```
素材源头：这篇的核心判断来自___（具体经历，不是思考）
翻案句：很多人以为___，但实际上___
不同意句：大多数读者读到这句话会想反驳，但无法否认：___
核心判断句：___（1 句，有立场，能独立成句）
行为改变句：读者读完后，___这个行为会改变或停止
```

不同意句的质量标准：

- 不是"大家都知道的道理换个说法"
- 不是"读者会觉得有道理"，而是"读者会觉得不舒服但否认不了"
- 如果你写完不同意句，自己感觉"读者应该不会反对"，这句话还不够锋利

同时输出：

- 写作风险（3 条，指出这篇最容易犯的具体错误）
- 文章骨架（按：表面问题 → 常见误判 → 底层机制 → 重新定价 → 可执行动作）

**五行任何一行写不出来，退回阶段 2。**

将五行写入状态文件 `core` 字段。

---

### 阶段 3：写作

品牌深度链路使用 `seo-content-writer`（不可用则手动完成）。
公众号快写链路使用 `wechat-article-writer`（不可用则手动完成）。

输入：

- `<workspace_root>/02_research/<slug>.md`
- `<workspace_root>/03_core/<slug>.md`（必须同时提供）

输出：`<workspace_root>/04_drafts/<slug>.md`

写作约束：

- 结论先行，不用空概念开头
- 先写读者已感受到的症状，再解释本质
- 不同意句必须出现在文章中（不能删掉，不能软化成没人反对的版本）
- 最强金句放在文章 60%-80% 位置，不顶在开头
- 结尾给裁决型标准，不用态度收尾
- 全文只讲一个核心判断

禁止词：赋能、闭环、未来已来、颠覆一切、行业洗牌、AI 时代来临

---

### 阶段 3.5：写后审查（强制）

使用 `knowledge-article-auditor`。

输入：`<workspace_root>/04_drafts/<slug>.md`

输出：审查报告

评分门槛：

- **16-20 分**：进入散文审查
- **12-15 分**：可进入散文审查，但必须标注待修复项
- **0-11 分**：退回写作阶段重写

额外检查项（auditor 评分之外）：

- 不同意句是否出现在文章中，且保留了锋利度（没有被软化）
- 文章实际素材来源是否可追溯（不是举例，是真实经历）

将审查分数写入状态文件 `audit_score` 字段。

---

### 阶段 3.6：散文审查（强制）

逐段检查文章，不检查结构，只检查每段文字是否在推进论点。

输入：`<workspace_root>/04_drafts/<slug>.md`

输出：散文审查报告，写回初稿文件末尾

检查每个段落，标记以下三类问题：

**类型 A：填充段**
这段说的是上一段已经说过的东西，只是换了说法。
标记方式：`[填充] 第 X 段 — 这段在重复第 Y 段的论点`

**类型 B：悬空段**
这段全是抽象词，没有任何具体锚点（场景、数字、行为、对话）。
判断标准：把这段里的名词全部删掉，剩下的骨架能不能单独成立。如果不能，这段在悬空。
标记方式：`[悬空] 第 X 段 — 抽象词：___，缺少的锚点类型：___`

**类型 C：滑行段**
这段没有向前推进，只是在把上一段说得更顺、更完整，但没有带出新信息。
标记方式：`[滑行] 第 X 段 — 删掉不影响论点推进`

审查完成后：

- 如果 A/B/C 类问题合计超过全文段落数的 30%，退回修稿，先处理散文问题
- 如果问题在 30% 以内，标注问题后继续推进，在格式优化阶段一并处理

将 `prose_cleared: true` 写入状态文件。

---

### 阶段 4：格式优化

使用 `baoyu-format-markdown`（不可用则手动完成）。

输入：`<workspace_root>/04_drafts/<slug>.md`

输出：`<workspace_root>/05_polish/<slug>.md`

格式优化时同步处理散文审查报告中标记的 A/B/C 类问题。

最低要求：

1. 标题必须是读者困惑型，不是作者总结型
2. 补齐摘要
3. 每个小标题描述一个具体论点，不是章节标签
4. 删除或重写散文审查标记的填充段、悬空段、滑行段
5. 补齐 CTA

---

### 阶段 4.5：入口优化（默认执行）

使用 `reader-hook`。

输入：`<workspace_root>/05_polish/<slug>.md`

输出：更新后的标题和开头，写回 `05_polish/<slug>.md`

检查标准：

- 标题是读者困惑，不是作者总结
- 开头先写症状，不先写本质
- 3 秒内能看懂这篇在说读者什么问题

如果当前标题和开头已满足以上三项，可跳过，说明原因。

---

### 阶段 4.9：完成确认（强制）

**在进入分发准备之前执行。**

必须能具体回答：

> 读者读完这篇，___ 这个行为会改变或停止。

合格答案标准：

- 指向具体行为，不是"会有更多思考"或"认知会提升"
- 合格示例：读者下次打开 AI 写第一句话时，会先停一下，想想这句话是不是自己真正要说的
- 不合格示例：读者会更重视判断力

如果无法写出合格答案，文章没有写完。退回阶段 3 重新检查核心判断是否落地。

完成确认后，将行为改变句写入状态文件 `core.behavior_change` 字段。

---

### 阶段 5：分发准备

使用 `baoyu-markdown-to-html`（不可用则手动完成）。

输入：`<workspace_root>/05_polish/<slug>.md`

输出：`<workspace_root>/06_publish/<slug>.html`

完成后暂停，汇报产物路径，等待用户确认是否发布。

---

### 阶段 6：发布（需用户明确确认）

只有用户明确说"发布"，才使用 `baoyu-post-to-wechat`。

输入：`<workspace_root>/06_publish/<slug>.html`

输出：已发布到微信公众号

## 重入诊断协议

当内容不够好需要修改时，不要直接重写初稿。

先做诊断，找到真正出问题的阶段，从那里重入。重写初稿是在修症状，不是在修根因。

### 第一步：症状 → 根因定位

根据"内容哪里不好"，对照下表定位根因和重入阶段：

| 症状 | 根因 | 重入阶段 |
|---|---|---|
| 文章读起来空、像在说道理、没有质感 | 素材不真实，从概念出发写的 | 阶段 -1 |
| 文章跑偏，跟品牌定位或四大支柱无关 | 品牌收束没做或没约束写作 | 阶段 0 |
| 文章有观点，但没有张力，读者不会不同意 | 不同意句不够锋利，或写作时被软化了 | 阶段 2.5 |
| 文章结构没问题，但段落读起来费力、拖沓 | 填充段、悬空段、滑行段过多 | 阶段 3.6 |
| 文章整体正确，但开头没有吸引力 | 入口问题，开头从本质而非症状开始 | 阶段 4.5 |
| 读者读完后没有行为改变，感觉"有道理但没用" | 核心判断没有落到可操作层 | 阶段 2.5 |
| 文章说了很多，但读完不知道中心是什么 | 内容核心没有收束，五行没写清楚 | 阶段 2.5 |
| 文章有场景，但场景是编的，读者感觉不真实 | 素材虚假，场景没有来自真实经历 | 阶段 -1 |
| 文章论点正确，但每段之间缺乏推进感 | 结构提纲问题，各段并列而非递进 | 阶段 2 |

如果症状同时符合多行，选最上游的那个。上游问题不解决，下游修了也会复发。

### 第二步：产物失效规则

重入某个阶段后，该阶段及之后的所有产物自动失效，必须重新生产。

### 第三步：快速通道

如果问题只在单一层，且上游阶段产物仍然成立，允许只修那一层，不重跑全流程。

**快速通道 A：只修入口**
适用症状：文章内容没问题，但标题和开头不够吸引人。
操作：直接重新执行阶段 4.5（入口优化），不重跑其他阶段。
前提：散文审查已通过，`prose_cleared: true`。

**快速通道 B：只修散文**
适用症状：论点和结构都对，但段落拖沓，读起来费力。
操作：直接重新执行阶段 3.6（散文审查）和阶段 4（格式优化）。
前提：写后审查分数 ≥ 16，内容核心五行仍然成立。

**快速通道 C：只修不同意句**
适用症状：文章有观点，但锋利度不足，读者读完没有被触动。
操作：回到 `03_core/<slug>.md`，只重写不同意句，然后检查初稿中对应段落是否需要同步修改。
前提：其余四行（素材源头、翻案句、核心判断句、行为改变句）仍然成立。

**快速通道 D：只修结尾**
适用症状：文章正文没问题，但结尾是态度收尾，没有裁决型标准。
操作：只重写结尾段，确认行为改变句落地，重新执行阶段 4.9（完成确认）。
前提：散文审查已通过，写后审查分数 ≥ 16。
