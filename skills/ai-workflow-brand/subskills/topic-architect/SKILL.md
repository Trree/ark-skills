# AI Workflow Topic Architect

先读取：

- `../../references/brand.md`
- `../../references/audience.md`
- `../../references/rules.md`

## 任务
生成：
- 选题
- 标题
- 栏目
- 系列规划

## 选题要求
题目必须命中四大支柱之一：

- **Judgment** — AI 为什么削弱判断，什么不能外包给 AI，如何保住判断
- **Method** — 什么经验值得沉淀，如何让方法可复用，skill/workflow/system 的边界
- **System** — 如何搭建个人系统，如何让 AI 成为系统部件而非大脑替代
- **Brand Asset** — 为什么写作是沉淀判断，如何让内容形成长期认知

## 每个选题必须明确
1. 属于哪个品牌支柱
2. 属于哪种使命类型（Define / Distinguish / Diagnose / Build / Sharpen）
3. 针对的是哪一个读者真实困境
4. 会强化读者对我的哪种认知

## 标题句式优先
- 为什么学了越多反而越乱？
- AI 帮你总结越多，你越没有自己的观点——这是怎么回事？
- 这不是工具问题，而是你没有内核
- 为什么你有 100 个 skill，却没有一套系统？
- 你在放大自己，还是在把自己蒸馏成流程？
- 判断力比效率更值钱，但你在保护哪一个？

## 避免
- XX 工具太强了
- 10 个必备神器
- 最全盘点
- 未来趋势分析
- 没有判断的经验分享

## 输出格式
1. 选题标题
2. 所属品牌支柱
3. 使命类型（Define / Distinguish / Diagnose / Build / Sharpen）
4. 核心读者困境
5. 本文核心判断
6. 可展开的 3 个论点
7. 本文要留下的模型或框架
8. 是否适合做系列
